Heroku

環境建置

安裝

  • node.js 版本須 > 8
  • 須安裝 npm
  • Getting Started on Heroku with Node.js

登入

heroku login

部署

創建 Git 與連結 App

git init
git add .
git commit -m 'first commit'

heroku create [app-name]
heroku git:remote -a [app-name]

git push heroku master

heroku 需要 start 指令才可建置專案，因此需於 package.json 裡加入:

"scripts": {
  "test": echo \"Error: no test specified\" && exit 1",
  "start": "node index.js"
},

修改專案名稱: heroku apps:rename --app [old-name] [new-name] 檢視遠端連結: git remote -v

Firebase

  • Firebase console
  • BaaS (Backend as a Service)

介紹

  1. Cloud Firestore
    ├ NoSQL Database
    ├ 範例：
    ├ iOS App (Swift): UhooiPicBook/MonstersRepository.swift
    ├ Android App (Kotlin): UhooiPicBook-Android/MonstersFirestoreClient.kt
    │ UhooiPicBook-Android/MonsterItem.kt
    └ Security Rule Test (Typescript): uhooi/UhooiPicBook-Firebase
  2. Cloud Functions
    └ 執行 serverless back-end code
  3. Authentication
    ├ 認證系統
    └ 實裝登入功能
  4. Hosting
    ├ Web Hosting
    └ 實裝 LP、PWA
  5. Cloud Storage
    ├ 雲端外部存放空間
    └ 儲存與提供圖片、影片檔案
  6. Realtime Database
    ├ NoSQL Database
    ├ 一般會選用 Firestore
    └ 差異參照：選擇一個數據庫：Cloud Firestore 或實時數據庫
  7. Crashlytics
    └ 檢視錯誤
  8. Performance Montioring
    └ 計測效能
  9. Test Lab
    ├ 裝置測試
    └ UI 測試、Monkey Test
  10. Google Analytics
    └ 分析使用者
  11. Predictions
  12. Cloud Messaging
    ├ 推送通知
    ├ 範例：
    │ Implement the foundation for remote push notifications #88
    └ Implement the foundation for remote push notifications #57
  13. Remote Config
    ├ 遠端變更 App 設定
    └ 期間限定外觀、A/B Test
  14. Dynamic Links
    ├ Deep Link
    └ 生成一連結，讓使用者已安裝則啟動 App，未安裝則移動至 Store

使用

  1. 新建 Project
  2. 從左上齒輪設定 Soruce Location
  3. 新增 App
  4. npm i -g firebase-tools
  5. 移動至作業資料夾
  6. npm i --save firebase
  7. firebase login
  8. firebase init
  9. firebase deploy
